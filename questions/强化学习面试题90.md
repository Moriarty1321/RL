# 强化学习训练中如何评价并保存最优模型

## 1. 模型评价指标

在强化学习训练过程中，需要选择合适的评价指标来判断模型性能。常见的评价指标包括：

- **平均回报（Average Return）**：评估智能体在环境中获得的累积奖励，是最直接的指标
- **成功率（Success Rate）**：适用于目标导向任务，计算完成任务的比例
- **步数效率（Step Efficiency）**：完成任务所需的平均步数，越少越好
- **稳定性（Stability）**：训练过程中性能波动程度

## 2. 评价方法

### 2.1 在线评估（Online Evaluation）

在训练过程中实时评估模型性能：

```python
def online_evaluation(agent, env, num_episodes=10):
    total_reward = 0
    for _ in range(num_episodes):
        state = env.reset()
        episode_reward = 0
        done = False
        while not done:
            action = agent.select_action(state)
            next_state, reward, done, _ = env.step(action)
            episode_reward += reward
            state = next_state
        total_reward += episode_reward
    return total_reward / num_episodes
```

### 2.2 离线评估（Offline Evaluation）

使用固定测试集进行评估，避免训练过程中的偏差：

- 创建独立的测试环境
- 固定随机种子以确保可复现性
- 多次运行取平均值

## 3. 最优模型保存策略

### 3.1 基于性能的保存

当模型性能超过历史最佳时保存：

```python
best_return = -float('inf')

for episode in range(total_episodes):
    # 训练过程...
    
    # 定期评估
    if episode % eval_freq == 0:
        current_return = evaluate(agent, env)
        
        if current_return > best_return:
            best_return = current_return
            torch.save(agent.state_dict(), 'best_model.pth')
            print(f"New best model saved with return: {best_return}")
```

### 3.2 滑动窗口平均

使用滑动窗口平均避免偶然性峰值：

```python
window_size = 10
return_window = []

for episode in range(total_episodes):
    # 训练过程...
    
    if episode % eval_freq == 0:
        current_return = evaluate(agent, env)
        return_window.append(current_return)
        
        if len(return_window) > window_size:
            return_window.pop(0)
            
        avg_return = sum(return_window) / len(return_window)
        
        if avg_return > best_avg_return:
            best_avg_return = avg_return
            save_model(agent, 'best_avg_model.pth')
```

## 4. 高级保存策略

### 4.1 多指标综合评估

结合多个指标进行综合判断：

$$
\text{Score} = w_1 \cdot \text{Return} + w_2 \cdot \text{SuccessRate} - w_3 \cdot \text{Steps}
$$

其中 $w_1, w_2, w_3$ 为权重系数，根据任务重要性调整。

### 4.2 早停机制（Early Stopping）

当性能在连续多个评估周期内未提升时停止训练：

```python
patience = 20
no_improvement_count = 0

for episode in range(total_episodes):
    # 训练和评估...
    
    if current_return > best_return:
        best_return = current_return
        no_improvement_count = 0
        save_model(agent, 'best_model.pth')
    else:
        no_improvement_count += 1
        
    if no_improvement_count >= patience:
        print("Early stopping triggered")
        break
```

## 5. 实践建议

- **评估频率**：建议每 100-1000 步评估一次，根据环境复杂度调整
- **评估次数**：每次评估运行 10-100 个 episode 以获得稳定估计
- **保存多个检查点**：除了最优模型，也保存定期检查点以便回滚
- **记录元数据**：保存模型时同时记录训练参数、评估指标等元信息

## 6. 注意事项

- 避免在训练环境中直接评估，应使用独立的测试环境
- 注意探索与利用的平衡，评估时应关闭探索（如设置 ε=0）
- 对于随机环境，确保评估时固定随机种子
- 考虑计算成本，过于频繁的评估会影响训练效率
