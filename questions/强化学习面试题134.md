# 强化学习面试题134：SAC算法中alpha_lr、actor_lr、critic_lr设置的值要一致吗？

SAC算法中alpha_lr、actor_lr、critic_lr设置的值要一致吗？

在Soft Actor-Critic (SAC) 算法中，**alpha_lr**、**actor_lr** 和 **critic_lr** 分别对应三个不同组件的学习率：

- **alpha_lr**：用于更新温度参数 $\alpha$（即熵正则化系数）的学习率。
- **actor_lr**：用于更新策略网络（Actor）的学习率。
- **critic_lr**：用于更新Q函数网络（Critic）的学习率。

这三个学习率**不需要设置为相同的值**。事实上，在实践中，它们通常被设置为**不同的值**，原因如下：

1. **不同组件的学习动态不同**：
   - Critic（Q网络）通常需要更稳定、更快的收敛，因此常使用相对较大的学习率。
   - Actor（策略网络）更新依赖于Critic的梯度，对学习率更敏感，通常使用较小或中等学习率以避免震荡。
   - Alpha（温度参数）控制探索与利用的权衡，其更新通常非常缓慢，因此 alpha_lr 一般设置得很小（例如 $10^{-4}$ 或更低）。

2. **经验调参建议**：
   - 常见配置示例：
- critic_lr: $3 \times 10^{-4}$
- actor_lr: $3 \times 10^{-4}$
- alpha_lr: $1 \times 10^{-4}$ 或 $3 \times 10^{-4}$
   - 有些实现中 alpha_lr 设为与 actor_lr 相同，但更多研究表明使用更小的 alpha_lr 有助于稳定训练。

3. **自动调节 alpha 的情况**：
   - 在SAC的原始论文及后续改进中， $\alpha$ 通常是自动调节的（通过优化目标 $\mathbb{E}_{a \sim \pi}[\alpha \log \pi(a|s) + \mathcal{H}_0]$），此时 alpha_lr 控制该优化过程的步长，应与策略和价值函数的学习率解耦。

✅ 总结：
> **alpha_lr、actor_lr、critic_lr 不需要一致，应根据各组件的学习特性独立设置。通常 critic_lr ≥ actor_lr > alpha_lr。**

参考配置（来自SAC官方实现或Stable Baselines3）：
```python
learning_rate: 3e-4          # 通常同时用于 actor 和 critic
# 但在分开设置时：
actor_lr: 3e-4
critic_lr: 3e-4
alpha_lr: 1e-4  # 或 3e-4，但更推荐较小值
```

> ⚠️ 注意：具体数值仍需根据任务环境和网络结构进行调参。
