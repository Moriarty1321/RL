# 强化学习面试题190：为什么PPO和SAC算法中计算entropy的方式不一样？

### 为什么PPO和SAC算法中计算entropy的方式不一样？

在强化学习中，PPO（Proximal Policy Optimization）和SAC（Soft Actor-Critic）都是广泛使用的策略优化算法，但它们在计算**策略熵（entropy）**的方式上存在显著差异。这种差异源于它们各自的理论基础、优化目标以及对探索（exploration）的处理方式。

---

#### 1. **PPO中的熵计算**

PPO是一种**on-policy**算法，其目标函数通常包含一个熵正则项（entropy bonus），用于鼓励探索：

$$
\mathcal{L}^{\text{PPO}}(\theta) = \mathbb{E}_t \left[ \min\left( r_t(\theta) \hat{A}_t, \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) \hat{A}_t \right) \right] + c_1 \mathcal{L}_t^{\text{value}} - c_2 \mathcal{S}[\pi_\theta](s_t)
$$

其中：

- $\mathcal{S}[\pi_\theta](s_t)$ 表示在状态 $s_t$ 下策略 $\pi_\theta$ 的熵。
- 对于**离散动作空间**，熵通常计算为：

$$
\mathcal{S}[\pi_\theta](s_t) = -\sum_{a} \pi_\theta(a|s_t) \log \pi_\theta(a|s_t)
$$

- 对于**连续动作空间**，若策略是高斯策略 $\pi_\theta(a|s) = \mathcal{N}(\mu_\theta(s), \sigma^2)$，则熵为：

$$
\mathcal{S}[\pi_\theta](s_t) = \frac{1}{2} \log(2\pi e \sigma^2)
$$

**特点：**

- 熵是作为**固定系数** $c_2$ 乘上的正则项，用于鼓励随机性。
- 熵项在训练过程中通常保持不变或缓慢衰减。
- PPO的熵计算是**显式添加**到目标函数中的，目的是防止策略过早收敛到确定性策略。

---

#### 2. **SAC中的熵计算**

SAC是一种**off-policy**算法，其核心思想是**最大熵强化学习（Maximum Entropy RL）**，即在最大化累积奖励的同时，也最大化策略的熵。

SAC的目标函数为：

$$
\mathbb{E} \left[ \sum_{t=0}^\infty \gamma^t \left( r(s_t, a_t) + \alpha \mathcal{H}(\pi(\cdot|s_t)) \right) \right]
$$

其中：

- $\mathcal{H}(\pi(\cdot|s_t))$ 是策略在状态 $s_t$ 下的熵。
- $\alpha$ 是**温度参数（temperature parameter）**，控制熵的权重。

对于**连续动作空间**，SAC通常使用**高斯策略**或**重参数化技巧（reparameterization trick）**，其熵为：

$$
\mathcal{H}(\pi(\cdot|s_t)) = \mathbb{E}_{a_t \sim \pi(\cdot|s_t)} \left[ -\log \pi(a_t|s_t) \right]
$$

在高斯策略下，若动作分布为 $\mathcal{N}(\mu_\theta(s), \sigma_\theta(s)^2)$，则：

$$
\mathcal{H}(\pi(\cdot|s_t)) = \frac{1}{2} \sum_{i=1}^d \left( \log(2\pi \sigma_i^2) + 1 \right)
$$

其中 $d$ 是动作维度。

**特点：**

- SAC将熵**直接融入奖励函数**，形成“软Q函数（soft Q-function）”：

$$
Q^{\text{soft}}(s, a) = r(s, a) + \gamma \mathbb{E}_{s^{\prime} \sim p(\cdot|s,a)} \left[ V^{\text{soft}}(s^{\prime}) \right]
$$

其中：

$$
V^{\text{soft}}(s^{\prime}) = \mathbb{E}_{a^{\prime} \sim \pi(\cdot|s^{\prime})} \left[ Q^{\text{soft}}(s^{\prime}, a^{\prime}) - \alpha \log \pi(a^{\prime}|s^{\prime}) \right]
$$

- 熵项是**动态计算**的，依赖于当前策略的概率密度。
- 温度参数 $\alpha$ 可以**自动调节**（如通过优化 $\alpha$ 使得平均熵接近目标熵）。

---

#### 3. **关键差异总结**

| 特性 | PPO | SAC |
|------|-----|-----|
| **算法类型** | On-policy | Off-policy |
| **熵的作用** | 正则项，防止过早收敛 | 目标函数的一部分，鼓励最大熵 |
| **熵计算方式** | 显式计算策略分布的熵 | 通过 $-\log \pi(a|s)$ 在采样中估计 |
| **熵的权重** | 固定或手动衰减的系数 $c_2$ | 可调节的温度参数 $\alpha$ |
| **理论基础** | 策略梯度 + 信任域 | 最大熵强化学习 |
| **动作空间** | 离散/连续均可 | 主要用于连续动作空间 |
| **熵的梯度传播** | 仅用于策略更新 | 影响Q函数和策略的联合优化 |

---

#### 4. **为什么不一样？**

根本原因在于**优化目标不同**：

- PPO 的目标是**最大化期望回报**，熵只是辅助探索的工具。
- SAC 的目标是**最大化熵加权的回报**，熵是优化目标的核心部分。

因此：

- PPO 中的熵是**加性正则项**，计算方式简单直接。
- SAC 中的熵是**内在奖励**，必须与Q函数和策略联合优化，计算更复杂，且依赖于策略的概率密度。

此外，SAC使用**重参数化**（如 $a = \mu + \sigma \cdot \epsilon$），使得 $-\log \pi(a|s)$ 可以高效计算并参与梯度传播，而PPO通常不依赖这种机制。

---

#### 5. **示例对比**

**PPO（连续动作）**：

```python
# 假设策略输出均值和标准差
mean, std = policy(state)
# 计算熵（高斯分布）
entropy = 0.5 * torch.sum(torch.log(2 * np.pi * std**2) + 1)
# 添加到损失
loss = policy_loss + value_loss - entropy_coeff * entropy
```

**SAC（连续动作）**：

```python
# 重参数化采样
eps = torch.randn_like(mean)
action = mean + std * eps
# 计算log_prob
log_prob = -0.5 * torch.sum((eps)**2 + 2 * torch.log(std) + np.log(2 * np.pi))
# 熵 = -log_prob（期望意义下）
# 用于软Q函数更新
soft_q_target = reward + gamma * (next_q - alpha * next_log_prob)
```

---

#### 结论

PPO和SAC在熵计算上的不同，本质上是由于：

- **PPO**：熵是**外部添加的正则项**，用于稳定训练和促进探索。
- **SAC**：熵是**优化目标的一部分**，通过最大熵框架自然融入学习过程。

这种差异反映了两种算法在**探索机制**和**策略优化哲学**上的根本不同。
