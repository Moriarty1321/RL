# 强化学习面试题119：强化学习SAC算法中，为什么在计算actor_loss的时候用的是alpha，但是optimizer却是针对log_alpha的？

在强化学习中的 Soft Actor-Critic (SAC) 算法中，关于为何在计算 `actor_loss` 时使用 $\alpha$，但优化器却针对 $\log \alpha$ 进行优化，可以从以下几个方面理解：

### 1. 数学上的等价性
SAC 算法的目标函数中包含熵正则项，其形式为：
$$
\mathbb{E}_{\pi_\theta} \left[ Q(s, a) - \alpha \log \pi_\theta(a|s) \right]
$$
其中 $\alpha$ 是温度参数（temperature parameter），用于权衡奖励与策略熵的重要性。

在实现中，我们通常希望 $\alpha > 0$，因为负的 $\alpha$ 会导致目标函数无意义（熵项被错误加权）。为了自动调整 $\alpha$，SAC 引入了一个关于 $\alpha$ 的优化目标：
$$
\min_\alpha \mathbb{E}_{a \sim \pi} \left[ -\alpha \log \pi(a|s) - \alpha \bar{\mathcal{H}} \right]
$$
其中 $\bar{\mathcal{H}}$ 是目标熵（target entropy），通常设为负值（如 $-\dim(\mathcal{A})$）。

由于 $\alpha$ 必须为正数，直接优化 $\alpha$ 需要施加约束（如投影到正实数域），这在数值优化中不够方便。

### 2. 使用 $\log \alpha$ 的原因：无约束优化
令：
$$
\log \alpha = \omega \quad \Rightarrow \quad \alpha = e^\omega
$$
通过将优化变量从 $\alpha$ 转换为 $\omega = \log \alpha$，我们可以：
- 将原本带约束的优化问题（$\alpha > 0$）转化为无约束优化（$\omega \in \mathbb{R}$）；
- 使用标准的梯度下降方法（如 Adam）直接更新 $\omega$，而无需担心违反正性约束；
- 在计算损失时，仍使用 $\alpha = e^\omega$ 来构造 `actor_loss` 或 `alpha_loss`。

因此，在代码中常见如下操作：
```python
alpha = self.log_alpha.exp()  # α = exp(log_α)
actor_loss = (alpha * log_pi - q_value).mean()
```
但优化器更新的是 `log_alpha`：
```python
self.alpha_optimizer.zero_grad()
alpha_loss.backward()
self.alpha_optimizer.step()  # 更新 log_alpha
```

### 3. 梯度传播的合理性
虽然 `actor_loss` 中显式使用了 $\alpha$，但 $\alpha$ 本身是通过 $\log \alpha$ 计算得到的（即 $\alpha = \exp(\log \alpha)$）。因此，当反向传播时，梯度会通过 $\alpha$ 自动传递到 $\log \alpha$ 上，无需额外处理。

换句话说，即使 `actor_loss` 中用的是 $\alpha$，只要 $\alpha$ 是由可训练的 `log_alpha` 参数通过 `exp` 得到的，PyTorch/TensorFlow 等框架就能正确计算 $\frac{\partial \text{actor\_loss}}{\partial \log \alpha}$，从而实现端到端训练。

### 4. 总结
- **使用 $\alpha$**：因为目标函数和熵正则项的定义天然涉及 $\alpha$；
- **优化 $\log \alpha$**：为了保证 $\alpha > 0$ 并实现无约束优化，提升数值稳定性与训练便利性。

这种做法是深度学习中处理正参数（如方差、温度系数等）的常见技巧，类似于在 VAE 中优化 $\log \sigma^2$ 而非 $\sigma^2$ 本身。
