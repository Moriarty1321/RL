# 强化学习面试题1：蒙特卡洛、TD、动态规划的关系？

# 蒙特卡洛、TD、动态规划的关系？

## 1. 动态规划（Dynamic Programming, DP）

- **前提条件**：需要**完整的环境模型**（即已知状态转移概率 $P(s^{\prime}|s,a)$ 和奖励函数 $R(s,a,s^{\prime})$）
- **更新方式**：基于**贝尔曼期望方程**进行**全宽回溯**（full backup）
- **特点**：
  - 每次更新都使用**当前策略下所有可能后继状态的期望值**
  - 属于**模型驱动**（model-based）方法
  - 可以**同步更新**所有状态的价值函数（如值迭代、策略迭代）
- **公式示例**（策略评估）：

$$
V_{k+1}(s) = \sum_{a} \pi(a|s) \sum_{s^{\prime}, r} P(s^{\prime}, r | s, a) \left[ r + \gamma V_k(s^{\prime}) \right]
$$

## 2. 蒙特卡洛方法（Monte Carlo, MC）

- **前提条件**：**无需环境模型**，仅需通过**完整轨迹**（episode）采样
- **更新方式**：基于**实际回报**（return）进行更新，只在 episode 结束后更新
- **特点**：
  - **无偏估计**：因为使用的是真实回报
  - **高方差**：由于回报依赖于整个轨迹的随机性
  - 属于**无模型**（model-free）方法
  - 只能用于**回合制任务**（episodic tasks）
- **公式示例**（首次访问 MC）：

$$
V(s) \leftarrow V(s) + \alpha \left[ G_t - V(s) \right]
$$

  其中 $G_t$ 是从时间步 $t$ 开始的实际回报。

## 3. 时序差分学习（Temporal Difference, TD）

- **前提条件**：**无需环境模型**，基于**部分轨迹**（单步或多步）进行更新
- **更新方式**：结合 DP 的**自举**（bootstrapping）思想和 MC 的**采样**思想
- **特点**：
  - **有偏但低方差**：因为使用了当前价值估计（自举），但基于采样数据
  - 属于**无模型**方法
  - 可用于**持续任务**（continuing tasks）
  - 在线更新（每步都可更新）
- **公式示例**（TD(0)）：

$$
V(s) \leftarrow V(s) + \alpha \left[ r + \gamma V(s^{\prime}) - V(s) \right]
$$

  其中 $r + \gamma V(s^{\prime})$ 称为 **TD 目标**，$r + \gamma V(s^{\prime}) - V(s)$ 称为 **TD 误差**。

## 4. 三者关系总结

| 特性 | 动态规划（DP） | 蒙特卡洛（MC） | 时序差分（TD） |
|------|----------------|----------------|----------------|
| 是否需要环境模型 | ✅ 是 | ❌ 否 | ❌ 否 |
| 更新时机 | 每步（可同步） | 每回合结束 | 每步 |
| 是否自举（bootstrapping） | ✅ 是 | ❌ 否 | ✅ 是 |
| 是否采样 | ❌ 否（全期望） | ✅ 是（完整轨迹） | ✅ 是（单步/多步） |
| 偏差 | 无偏（若模型准确） | 无偏 | 有偏 |
| 方差 | 低 | 高 | 低 |
| 适用任务类型 | 任意 | 回合制 | 任意（包括持续任务） |

> **核心思想演进**：
> - DP 使用**期望更新** + **自举**
> - MC 使用**采样回报** + **无自举**
> - TD 使用**采样回报** + **自举** → **结合了 DP 和 MC 的优点**

- **TD 是 DP 与 MC 的桥梁**：  
  当 TD 的步长趋于完整轨迹时，TD 趋近于 MC；  
  当环境模型已知且 TD 使用真实期望时，TD 趋近于 DP。

- **统一视角**：  
  三者均可视为对贝尔曼方程的不同近似方式：

$$
V(s) = \mathbb{E}_\pi \left[ R_{t+1} + \gamma V(S_{t+1}) \mid S_t = s \right]
$$

  - DP：直接计算期望（需模型）
  - MC：用完整回报 $G_t$ 近似右边
  - TD：用单步回报 $R_{t+1} + \gamma V(S_{t+1})$ 近似右边

## 5. 扩展：n 步 TD 与 TD($\lambda$)

- **n 步 TD**：结合 n 步实际回报与自举

$$
G_t^{(n)} = R_{t+1} + \gamma R_{t+2} + \cdots + \gamma^{n-1} R_{t+n} + \gamma^n V(S_{t+n})
$$

- **TD($\lambda$)**：对 n 步回报进行加权平均（指数衰减权重）

$$
G_t^\lambda = (1 - \lambda) \sum_{n=1}^{\infty} \lambda^{n-1} G_t^{(n)}
$$

  - 当 $\lambda = 0$：退化为 TD(0)
  - 当 $\lambda = 1$：退化为 MC（在回合制任务中）

> 因此，**TD($\lambda$)** 提供了一个从 TD 到 MC 的连续过渡。
