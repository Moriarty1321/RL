# 多臂老虎机与强化学习算法的差别

## 1. 问题背景

多臂老虎机（Multi-Armed Bandit, MAB）问题和强化学习（Reinforcement Learning, RL）都是用于解决**序贯决策**（sequential decision making）问题的框架。尽管两者在思想上具有相似性，但它们在建模方式、目标设定以及适用范围上存在关键差异。

---

## 2. 多臂老虎机（Multi-Armed Bandit）

### 2.1 基本设定

- 智能体（Agent）在每一轮 $ t = 1, 2, \dots, T $ 中选择一个动作（即“拉杆”）$ a_t \in \mathcal{A} $，其中 $ |\mathcal{A}| = K $ 表示有 $ K $ 个可选的臂。
- 执行动作后，智能体获得一个**即时奖励** $ r_t \sim P(r|a_t) $，该奖励仅依赖于当前选择的动作。
- **无状态转移**：环境没有内部状态，或状态不随动作改变；每次决策相互独立。
- 目标是**最小化累积遗憾**（cumulative regret）：
  ```math
  R(T) = \sum_{t=1}^T \left( \mu^* - \mu_{a_t} \right)
  ```
  其中 $ \mu^* = \max_{a \in \mathcal{A}} \mathbb{E}[r|a] $ 是最优臂的期望奖励，$ \mu_{a_t} $ 是所选臂的期望奖励。

### 2.2 特点

- **无状态性**：不涉及状态转移或环境动态。
- **单步决策**：每次决策只影响当前奖励，不影响未来可选动作或状态。
- **探索-利用权衡**（Exploration-Exploitation Trade-off）是核心挑战。
- 典型算法：$\epsilon$-greedy、UCB（Upper Confidence Bound）、Thompson Sampling。

---

## 3. 强化学习（Reinforcement Learning）

### 3.1 基本设定

- 智能体与环境交互，在**离散时间步** $ t = 0, 1, 2, \dots $ 中进行决策。
- 在时刻 $ t $，智能体处于状态 $ s_t \in \mathcal{S} $，选择动作 $ a_t \in \mathcal{A} $。
- 执行动作后，环境转移到新状态 $ s_{t+1} \sim P(s_{t+1}|s_t, a_t) $，并返回奖励 $ r_t \sim P(r|s_t, a_t) $。
- 目标是**最大化期望累积奖励**（通常带折扣）：
  ```math
  \mathbb{E} \left[ \sum_{t=0}^\infty \gamma^t r_t \right], \quad 0 \leq \gamma \leq 1
  ```
  其中 $ \gamma $ 是折扣因子。

### 3.2 特点

- **状态依赖性**：奖励和转移依赖于当前状态和动作。
- **长期规划**：当前动作可能影响未来状态和奖励，需考虑长期回报。
- 建模为马尔可夫决策过程（MDP）：$ (\mathcal{S}, \mathcal{A}, P, R, \gamma) $。
- 典型算法：Q-learning、SARSA、Policy Gradient、Actor-Critic、Deep Q-Network (DQN) 等。

---

## 4. 核心差异对比

| 维度 | 多臂老虎机（MAB） | 强化学习（RL） |
|------|------------------|---------------|
| **状态空间** | 无状态或状态不变 | 有状态，且状态随动作转移 |
| **决策影响** | 仅影响当前奖励 | 影响未来状态和奖励 |
| **目标函数** | 最小化累积遗憾 | 最大化期望累积奖励 |
| **环境动态** | 静态（奖励分布固定） | 动态（状态转移 + 奖励函数） |
| **建模框架** | 非马尔可夫（或无状态 MDP） | 马尔可夫决策过程（MDP） |
| **算法复杂度** | 较低（仅需维护动作价值估计） | 较高（需处理状态-动作价值函数或策略） |
| **适用场景** | A/B 测试、广告推荐、临床试验 | 游戏 AI、机器人控制、自动驾驶 |

> **注**：多臂老虎机可视为强化学习在**状态空间为单点**（即只有一个状态）时的特例。

---

## 5. 示例说明

### 5.1 MAB 示例：在线广告选择

- 有 3 个广告候选（臂），每次展示一个给用户。
- 用户点击则获得奖励 1，否则为 0。
- 点击率固定，不随历史展示变化。
- 目标：快速识别点击率最高的广告并多展示它。

✅ 适用 MAB（如 UCB 算法）。

### 5.2 RL 示例：自动驾驶

- 车辆处于不同路况（状态：位置、速度、交通灯等）。
- 选择动作（加速、刹车、转向）。
- 动作影响下一时刻的状态和安全性/效率奖励。
- 需学习策略以在复杂环境中安全高效行驶。

✅ 必须使用强化学习（如 DQN 或 PPO）。

---

## 6. 总结

- **多臂老虎机** 是**无状态、单步奖励最大化**的简化决策问题，聚焦于探索与利用的平衡。
- **强化学习** 是**有状态、长期回报优化**的通用框架，能够处理复杂的序贯决策任务。
- MAB 可看作 RL 的**退化情形**，但两者在理论和算法设计上存在显著差异。

> 在实际应用中，应根据问题是否具有状态转移和长期依赖来选择合适的框架。
