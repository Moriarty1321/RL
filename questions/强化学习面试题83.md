# 强化学习中的奖励标准化

## 1. 引言

在强化学习中，**奖励信号**（reward signal）是智能体学习策略的核心驱动力。然而，原始奖励往往存在尺度差异大、稀疏性或分布不稳定等问题，这会导致训练过程不稳定或收敛困难。为了解决这些问题，**奖励标准化**（Reward Normalization）被广泛采用。

---

## 2. 为什么需要奖励标准化？

- **梯度不稳定**：奖励值过大或过小会导致策略梯度估计的方差过高，影响优化稳定性。
- **不同任务间奖励尺度不一致**：例如，某些环境奖励范围是 $[0, 1]$，而另一些可能是 $[-1000, 1000]$，直接比较或迁移学习困难。
- **加速收敛**：标准化后的奖励有助于优化器更有效地更新参数。

---

## 3. 常见的奖励标准化方法

### 3.1 移动平均标准化（Moving Average Normalization）

使用指数移动平均（Exponential Moving Average, EMA）对奖励进行归一化：

$$
\hat{r}_t = \frac{r_t - \mu_t}{\sigma_t + \epsilon}
$$

其中：
- $ \mu_t $ 是奖励的指数移动平均值：  

$$
\mu_t = \beta \mu_{t-1} + (1 - \beta) r_t
$$

- $ \sigma_t $ 是奖励的标准差估计（也可用 EMA 计算）
- $ \epsilon $ 是小常数（如 $10^{-8}$），防止除零

> **优点**：在线计算，适用于非平稳环境  
> **缺点**：对初始值敏感，可能引入偏差

---

### 3.2 回报标准化（Return Normalization）

对每个 episode 的**总回报**（return）进行标准化：

$$
G_t = \sum_{k=0}^{T-t-1} \gamma^k r_{t+k}
$$

然后对一批 episode 的 $ G_t $ 计算均值和标准差：

$$
\hat{G}_t = \frac{G_t - \bar{G}}{\sigma_G + \epsilon}
$$

> **适用场景**：A2C、PPO 等基于优势函数的算法  
> **注意**：需在每个 batch 或 episode 结束后进行，不能完全在线

---

### 3.3 奖励裁剪（Reward Clipping）

虽然不是严格意义上的标准化，但常与标准化结合使用：

$$
r_t^{\text{clipped}} = \text{clip}(r_t, -c, c)
$$

其中 $ c $ 是裁剪阈值（如 $ c = 1 $）

> **用途**：防止极端奖励值破坏训练  
> **典型应用**：DQN 中使用 $ r_t \in [-1, 1] $

---

## 4. 实践建议

- **优先使用移动平均标准化**：适用于大多数 on-policy 和 off-policy 算法
- **结合优势函数标准化**：在 PPO 中，通常对优势函数 $ A(s,a) $ 也进行 batch 标准化
- **避免在测试阶段使用标准化**：标准化参数（如 $\mu, \sigma$）应在训练时学习，测试时固定或关闭
- **监控奖励分布**：可视化奖励的均值和方差，确保标准化有效

---

## 5. 代码示例（PyTorch 风格）

```python
class RewardNormalizer:
    def __init__(self, beta=0.999, eps=1e-8):
        self.beta = beta
        self.eps = eps
        self.mu = 0.0
        self.var = 1.0
        self.count = 0

    def update(self, reward):
        self.count += 1
        delta = reward - self.mu
        self.mu += (1 - self.beta) * delta
        delta2 = reward - self.mu
        self.var = self.beta * self.var + (1 - self.beta) * delta * delta2

    def normalize(self, reward):
        std = (self.var + self.eps) ** 0.5
        return (reward - self.mu) / std
```

---

## 6. 总结

| 方法 | 在线性 | 适用算法 | 优点 | 缺点 |
|------|--------|----------|------|------|
| 移动平均标准化 | ✅ | PPO, A2C | 实时更新，稳定 | 初始偏差 |
| 回报标准化 | ❌ | A2C, TRPO | 更准确 | 需完整 episode |
| 奖励裁剪 | ✅ | DQN, PPO | 简单有效 | 丢失信息 |

> **关键点**：奖励标准化不是银弹，需结合具体任务和算法选择合适策略。

---
