# 强化学习 Actor-Critic 网络设计原则与激活函数选择

## 1. 引言

Actor-Critic 方法是强化学习中一类结合了值函数估计（Critic）和策略优化（Actor）的算法框架。其核心思想是同时学习两个组件：

- **Actor**：负责选择动作，即策略函数 $\pi(a|s; \theta)$，其中 $\theta$ 是策略网络参数。
- **Critic**：评估当前策略的好坏，即价值函数 $V(s; w)$ 或 $Q(s,a; w)$，其中 $w$ 是价值网络参数。

通过两者的协同训练，Actor-Critic 方法能够在高维连续动作空间中实现高效策略学习。

## 2. 网络结构设计原则

### 2.1 共享特征提取层（可选）

对于状态空间维度较高（如图像输入）的任务，可设计共享底层网络提取特征：

```
Input State s
    ↓
[Shared Feature Extractor] → e.g., CNN for images / MLP for vectors
    ↙                     ↘
[Actor Head]           [Critic Head]
    ↓                     ↓
π(a|s; θ)             V(s; w)
```

> **注意**：是否共享底层取决于任务复杂度与训练稳定性需求。共享可减少参数量，但可能导致梯度冲突。

### 2.2 独立网络设计

在大多数实现中，Actor 和 Critic 使用**独立的神经网络**，以避免梯度干扰：

- **Actor 网络**：输出动作分布（离散动作用 softmax，连续动作用高斯分布的均值 $\mu(s)$ 和方差 $\sigma^2(s)$）。
- **Critic 网络**：输出状态价值 $V(s)$ 或动作价值 $Q(s,a)$。

### 2.3 输出层设计

| 网络类型 | 动作空间 | 输出层设计 |
|--------|--------|----------|
| Actor  | 离散   | 全连接层 + Softmax → 概率分布 |
| Actor  | 连续   | 全连接层 → $\mu(s)$；另设可学习参数或对数方差 $\log \sigma^2(s)$ |
| Critic | 任意   | 单神经元线性输出 → $V(s)$ 或 $Q(s,a)$ |

> 对于连续动作空间，通常假设动作服从高斯分布：

```math
a \sim \mathcal{N}(\mu(s; \theta), \sigma^2(s; \theta))
```

## 3. 激活函数选择指南

激活函数的选择直接影响梯度流动、收敛速度和表达能力。

### 3.1 隐藏层推荐

- **ReLU（Rectified Linear Unit）**：  
  $f(x) = \max(0, x)$  
  优点：计算高效，缓解梯度消失；广泛用于中等复杂度任务。

- **Leaky ReLU**：  
  ```math
  f(x) = \begin{cases} x & \text{if } x > 0 \\ \alpha x & \text{otherwise} \end{cases}, \quad \alpha \in (0,1)
  ```
  优点：避免神经元“死亡”，适合深层网络。

- **Tanh**：  
  $f(x) = \tanh(x)$  
  优点：输出零中心化，适合需要对称激活的场景（如策略均值输出前）。

> ✅ **推荐组合**：隐藏层使用 **Leaky ReLU** 或 **Tanh**，尤其当状态归一化良好时。

### 3.2 输出层激活函数

| 网络 | 输出类型 | 激活函数 | 说明 |
|------|--------|--------|------|
| Actor（离散） | 动作概率 | **Softmax** | 保证概率和为 1 |
| Actor（连续） | 均值 $\mu$ | **None 或 Tanh** | 若动作有界（如 [-1,1]），用 Tanh 约束输出 |
| Actor（连续） | 方差 $\sigma^2$ | **Softplus 或 Exp** | 保证正值：$\sigma^2 = \log(1 + e^x)$ 或 $e^x$ |
| Critic | 价值 $V(s)$ | **Linear（无激活）** | 价值可为任意实数 |

> ⚠️ 避免在 Critic 输出层使用 ReLU/Tanh 等非线性激活，否则会限制价值范围，导致偏差。

### 3.3 特殊注意事项

- **梯度爆炸/消失**：深层网络建议使用 **Layer Normalization** 或 **Batch Normalization** 配合 Leaky ReLU。
- **探索与稳定性**：在 Actor 的方差输出中，使用 **Softplus** 而非 Exp，可避免数值溢出。
- **归一化输入**：确保状态输入经过标准化（均值 0，方差 1），有助于激活函数工作在其敏感区间。

## 4. 典型架构示例（连续控制）

```python
# Pseudo-code for Actor-Critic with continuous actions
class Actor(nn.Module):
    def __init__(self, state_dim, action_dim):
        super().__init__()
        self.fc1 = nn.Linear(state_dim, 256)
        self.fc2 = nn.Linear(256, 256)
        self.mu = nn.Linear(256, action_dim)
        self.log_std = nn.Parameter(torch.zeros(action_dim))  # 可学习对数标准差

    def forward(self, state):
        x = F.leaky_relu(self.fc1(state))
        x = F.leaky_relu(self.fc2(x))
        mu = torch.tanh(self.mu(x))  # 假设动作范围 [-1, 1]
        std = torch.exp(self.log_std)
        return mu, std

class Critic(nn.Module):
    def __init__(self, state_dim):
        super().__init__()
        self.fc1 = nn.Linear(state_dim, 256)
        self.fc2 = nn.Linear(256, 256)
        self.value = nn.Linear(256, 1)

    def forward(self, state):
        x = F.leaky_relu(self.fc1(state))
        x = F.leaky_relu(self.fc2(x))
        v = self.value(x)  # 线性输出
        return v
```

## 5. 总结

- Actor 和 Critic 网络应**独立设计**，避免梯度冲突。
- 隐藏层优先选用 **Leaky ReLU** 或 **Tanh**。
- Actor 输出需根据动作空间类型选择合适的激活函数（Softmax / Tanh + Softplus）。
- Critic 输出必须为**线性激活**，以准确估计价值。
- 输入归一化与适当的权重初始化（如 Xavier）对训练稳定性至关重要。

> 正确的设计原则与激活函数选择是 Actor-Critic 算法稳定收敛的关键前提。
