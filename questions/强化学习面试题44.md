# 强化学习面试题44：SAC算法中的重参数化，以及为什么引入重参数化？

**SAC算法中的重参数化，以及为什么引入重参数化？**

---

### 1. SAC 算法简介

Soft Actor-Critic (SAC) 是一种基于最大熵强化学习框架的 off-policy 算法，其目标是最大化期望回报的同时，也最大化策略的熵：

```math
J(\pi) = \mathbb{E}_{\tau \sim \pi} \left[ \sum_{t=0}^{\infty} \gamma^t \left( r(s_t, a_t) + \alpha \mathcal{H}(\pi(\cdot|s_t)) \right) \right]
```

其中：
- $ \mathcal{H}(\pi(\cdot|s)) = -\log \pi(a|s) $ 是策略在状态 $ s $ 下的熵；
- $ \alpha $ 是温度参数，控制熵正则化的强度。

SAC 使用 actor-critic 架构，其中：
- **Actor**：参数化策略 $ \pi_\theta(a|s) $，通常建模为高斯分布；
- **Critic**：学习 Q 函数 $ Q_\phi(s, a) $，用于评估动作价值。

---

### 2. 策略的参数化形式

在 SAC 中，策略通常被建模为一个**带可学习方差的高斯策略**：

```math
a \sim \pi_\theta(a|s) = \mathcal{N}(a|\mu_\theta(s), \sigma_\theta(s))
```

为了支持随机性并便于梯度传播，SAC 引入了**重参数化技巧（Reparameterization Trick）**。

---

### 3. 重参数化技巧

重参数化的核心思想是：将随机性从计算图中“抽离”出来，使得采样过程可导。

原始采样方式（不可导）：
```math
a \sim \pi_\theta(a|s)
```

重参数化后：
```math
\epsilon \sim \mathcal{N}(0, I), \quad a = f_\theta(s, \epsilon) = \mu_\theta(s) + \sigma_\theta(s) \odot \epsilon
```

其中 $ \odot $ 表示逐元素乘法。

这样，动作 $ a $ 成为关于参数 $ \theta $ 和独立噪声 $ \epsilon $ 的确定性函数，从而可以通过链式法则对 $ \theta $ 求梯度。

---

### 4. 为什么引入重参数化？

#### 4.1 梯度估计的方差降低

若直接对策略梯度使用**似然比梯度（REINFORCE-style）**：

```math
\nabla_\theta \mathbb{E}_{a \sim \pi_\theta} [Q(s,a)] = \mathbb{E}_{a \sim \pi_\theta} [Q(s,a) \nabla_\theta \log \pi_\theta(a|s)]
```

该估计器虽然无偏，但**方差较高**，尤其是在高维动作空间中，训练不稳定。

而使用重参数化后，梯度变为：

```math
\nabla_\theta \mathbb{E}_{\epsilon \sim p(\epsilon)} [Q(s, f_\theta(s, \epsilon))] = \mathbb{E}_{\epsilon \sim p(\epsilon)} [\nabla_\theta Q(s, f_\theta(s, \epsilon))]
```

由于梯度通过 $ f_\theta $ 传播，利用了 Q 函数的平滑性，**梯度估计方差更低**，训练更稳定。

#### 4.2 支持 off-policy 学习

SAC 是 off-policy 算法，使用回放缓冲区（replay buffer）存储历史转移 $ (s, a, r, s^{\prime}) $。

若动作 $ a $ 是通过不可导的采样得到，则无法直接使用这些 off-policy 数据更新策略。

重参数化使得策略更新可以基于回放缓冲区中的任意 $ (s, a) $ 对进行，只需将 $ a $ 表示为 $ f_\theta(s, \epsilon) $ 的形式，即可计算梯度。

> 实际上，SAC 在更新 actor 时，会重新采样 $ \epsilon $ 来生成新动作，而不是直接使用 buffer 中的 $ a $。

#### 4.3 与自动微分兼容

现代深度学习框架（如 PyTorch、TensorFlow）依赖自动微分。

重参数化将采样过程转化为确定性计算图，使得梯度可以通过反向传播自动计算，极大简化实现。

---

### 5. SAC 中的具体实现

在 SAC 的策略网络中：

```python
def forward(self, state):
    mean = self.mean_layer(state)
    log_std = self.log_std_layer(state)
    std = torch.exp(log_std)
    normal = Normal(mean, std)
    x_t = normal.rsample()  # 重参数化采样
    action = torch.tanh(x_t)
    return action
```

其中：
- `rsample()` 表示重参数化采样；
- 使用 $ \tanh $ 将高斯样本映射到有界动作空间（如 $[-1, 1]$）；
- 通过 $ \log(1 - \tanh^2(x)) $ 修正概率密度（用于熵计算）。

---

### 6. 总结

| 问题 | 回答 |
|------|------|
| **什么是重参数化？** | 将随机采样表示为确定性函数 + 外部噪声，使采样过程可导。 |
| **SAC 中如何使用？** | $ a = \mu_\theta(s) + \sigma_\theta(s) \odot \epsilon $，$ \epsilon \sim \mathcal{N}(0,I) $ |
| **为什么引入？** | 降低梯度估计方差、支持 off-policy 学习、兼容自动微分。 |

重参数化是 SAC 实现高效、稳定训练的关键技术之一。
